## Component Inventory

### 1. `<header>` — Sticky Site Header

**Behavior:** Fixed to top on scroll. Full-width. Black background.

| Property | Value |
|----------|-------|
| Height (desktop) | 72px — accommodates logo + nav + breathing room |
| Height (mobile) | 60px |
| Background | `--color-black` |
| Position | `position: sticky; top: 0; z-index: 100` |
| Shadow | `box-shadow: 0 2px 8px rgba(0,0,0,0.3)` — ensures header lifts above content on scroll |

**Internal layout (desktop, ≥769px):**
- Left: Logo (`<img src="logo.jpg" alt="Paradigm Inspection logo">`) — max-height 48px, auto width
- Center-right: Phone number as `<a href="tel:[YOUR-PHONE]">` — Inter 14px weight 500, color `--color-orange`
- Right: Nav links row — Inter 16px weight 400, color `#FFFFFF`, hover color `--color-orange` with 2px bottom border

**Internal layout (mobile, ≤768px):**
- Left: Logo
- Right: Hamburger button (replaces nav links row)
- Phone number moves into hamburger menu as first item

**CSS layout:** `display: flex; align-items: center; justify-content: space-between; padding: 0 var(--space-lg)` at all breakpoints. Max content width: 1200px centered inside.

---

### 2. `<nav>` — Navigation Links

**Desktop (≥769px):**
- Inline flex row, `gap: var(--space-xl)`
- Each link: `<a href="#section-id">Label</a>`
- Font: Inter 16px weight 400
- Color: `#FFFFFF`
- Hover: color transitions to `--color-orange`, `border-bottom: 2px solid var(--color-orange)`
- Transition: `color 200ms ease, border-bottom-color 200ms ease`
- Touch target: minimum 44px height via `padding: var(--space-sm) var(--space-xs)`

**Mobile (≤768px):**
- Nav links hidden by default: `display: none` (toggled by JS)
- When open: displayed as vertical stack, `display: flex; flex-direction: column`
- Dropdown attaches below header, full-width, black background, cream text
- Each mobile link: `padding: var(--space-md) var(--space-lg)` — ensures ≥44px touch target height
- Phone number appears as first item in mobile menu, styled in orange

---

### 3. Hamburger Button

**Visible:** only at ≤768px

**Appearance:**
- Three horizontal bars (SVG inline or `<span>` bars) — NOT a Lucide icon (Lucide requires JS load; SVG inline is zero-dependency)
- Color: `#FFFFFF`, hover color `--color-orange`
- Size: 24×18px bar group; button element is 44×44px minimum (touch target)
- Background: transparent, no border

**Interaction:**
- On click: toggles `.nav-open` class on `<header>` or `<nav>`
- JS adds `aria-expanded="true/false"` on the button
- Mobile menu slides in or appears — use `display: none / flex` toggle (no animation framework); optionally add `transition: opacity 200ms ease` for a soft fade
- Button has `aria-label="Open navigation menu"` (changes to "Close navigation menu" when open)
- Button has `aria-controls="main-nav"` pointing to the nav element's `id`

**SVG markup (inline, zero-dependency):**
```html
<button class="nav-toggle" aria-label="Open navigation menu" aria-expanded="false" aria-controls="main-nav">
  <svg width="24" height="18" viewBox="0 0 24 18" fill="none" xmlns="http://www.w3.org/2000/svg" aria-hidden="true">
    <rect y="0" width="24" height="2" rx="1" fill="currentColor"/>
    <rect y="8" width="24" height="2" rx="1" fill="currentColor"/>
    <rect y="16" width="24" height="2" rx="1" fill="currentColor"/>
  </svg>
</button>
```

## Interaction Contracts

| Interaction | Trigger | Response | Duration |
|------------|---------|----------|---------|
| Hamburger open | Click `.nav-toggle` when closed | Add `.nav-open` to `<header>`; nav becomes `display: flex`; `aria-expanded` → `"true"`; `aria-label` → `"Close navigation menu"` | immediate (optional 200ms fade) |
| Hamburger close | Click `.nav-toggle` when open | Remove `.nav-open`; nav becomes `display: none`; `aria-expanded` → `"false"`; `aria-label` → `"Open navigation menu"` | immediate (optional 200ms fade) |
| Nav link click (mobile) | Click any anchor link in open mobile menu | Close mobile menu (same as hamburger close), then browser scrolls to anchor | close immediate, scroll native |
| Nav link hover (desktop) | Hover over anchor link | Color → `--color-orange`; `border-bottom: 2px solid var(--color-orange)` appears | 200ms ease |
| Phone link hover | Hover over phone `<a>` in header | Underline appears | 200ms ease |
| Smooth scroll | Any anchor `href="#section"` clicked | Native CSS smooth scroll: `scroll-behavior: smooth` on `<html>` — no JS needed | browser-native |

## Accessibility Contract

| Requirement | Implementation |
|------------|---------------|
| Touch targets ≥ 44x44px | Hamburger button: `min-width: 44px; min-height: 44px`. Nav links: `padding` ensures 44px tap height. Enforces SITE-07. |
| Focus visible | Do not suppress `:focus-visible` outline. Use default browser outline or add `outline: 2px solid var(--color-orange); outline-offset: 2px` |
| Hamburger semantics | `<button>` element (not `<div>`), `aria-expanded`, `aria-controls`, `aria-label` toggled by JS |
| Screen reader nav skip | Not required for Phase 1 minimum (deferred to Phase 2 review) |
| Color contrast | Black (`#0A0A0A`) on cream (`#F5EDD6`) — contrast ratio ~15:1, well above WCAG AA 4.5:1 |
| Orange on black (phone/accent) | `#E07A2F` on `#0A0A0A` — contrast ratio ~4.6:1, meets WCAG AA for normal text at 16px+ |
