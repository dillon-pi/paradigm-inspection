**Create `script.js`** at the project root. Vanilla JS only, under 50 lines per the UI-SPEC File Structure Contract.

The script handles:

1. **Hamburger toggle** (per NAV-03, UI-SPEC Interaction Contracts):
   ```javascript
   (function () {
     'use strict';

     var toggle = document.querySelector('.nav-toggle');
     var header = document.querySelector('.site-header');

     if (!toggle || !header) return;

     toggle.addEventListener('click', function () {
       var isOpen = header.classList.toggle('nav-open');
       toggle.setAttribute('aria-expanded', String(isOpen));
       toggle.setAttribute('aria-label',
         isOpen ? 'Close navigation menu' : 'Open navigation menu'
       );
     });

     // Close mobile menu when any nav anchor link is clicked (per UI-SPEC)
     var navLinks = document.querySelectorAll('#main-nav a[href^="#"]');
     navLinks.forEach(function (link) {
       link.addEventListener('click', function () {
         header.classList.remove('nav-open');
         toggle.setAttribute('aria-expanded', 'false');
         toggle.setAttribute('aria-label', 'Open navigation menu');
       });
     });
   })();
   ```

Key requirements for the script:
- Wrapped in an IIFE — no global scope pollution
- `'use strict'` mode
- Uses `classList.toggle` which returns boolean for aria-expanded
- Anchor link clicks inside mobile nav close the menu per UI-SPEC interaction contract
- aria-expanded toggles between "true" and "false" strings
- aria-label toggles between "Open navigation menu" and "Close navigation menu"
- No external dependencies, no ES module syntax (plain script tag load)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-01-02 | Information Disclosure | tel: link placeholder | accept | Phone placeholder is bracketed; no real PII exposed until owner replaces it. Owner is instructed to replace before launch. |
| T-01-03 | Spoofing | 404.html | accept | Static site has no authentication or user sessions. 404 page is informational only. |
| T-01-04 | Elevation of Privilege | script.js | mitigate | Script is minimal (hamburger toggle only), uses strict mode, IIFE-scoped, no eval/innerHTML/dynamic script injection. No user input is processed. |
| T-01-05 | Tampering | .nojekyll | accept | Empty file; tampering would only re-enable Jekyll processing, which would be caught by deployment verification (site returns Jekyll error instead of content). |
